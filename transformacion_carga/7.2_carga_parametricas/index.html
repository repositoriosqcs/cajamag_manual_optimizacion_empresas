    Carga de Paramétricas¶

    Proceso de Carga de Información Paramétrica a DWH¶

    De acuerdo con la tercera regla definida, el proceso de subir la información paramétrica a la base DWH depende de las tablas identificadas en la Tabla 2 y que están marcadas con “Sí” en la columna ¿Requiere Carga en dwh? (con excepción de la tabla Madurez).

    La herramienta definida inicialmente para subir los datos fue MySQL Workbench. Sin embargo, debido al volumen de información manejado, se evidenció que MySQL Workbench presenta tiempos más altos para procesar los datos. Por ejemplo, cargar información de 70 mil registros toma cerca de 45 minutos, mientras que utilizando Python, el tiempo se reduce a menos de 2 minutos.

    A continuación, se presentan las dos opciones para subir la información a la base de datos DWH:

    Opción 1: Preparación usando MySQL Workbench¶

    Fuente 6: ZoneDimCoor¶

    Esta fuente corresponde a información de coordenadas para las zonas suministradas por Cajamag. Este origen ya está en DWH con el nombre dwh.Coordenadas.
    Condición: No es necesario actualizarlo, a menos que se cree una nueva zona. En ese caso, siga los pasos a continuación:

    1. Tomar el archivo “Coordenadas.xlsx” (revisar punto 4 de la sección 7.1) y agregar la nueva información.
    2. Guardar como CSV.
    3. Reemplazar la tabla dwh.Coordenadas en DWH subiendo la nueva información.

    Fuente 8: Apo_Proy¶

    Esta fuente corresponde a la información de proyección de aportes extraída inicialmente del documento “Id 114. Cumplimiento de los aportes”. Este origen ya está en DWH con el nombre dwh.Proyeccion_cumpl_aportes.
    Condición: Es necesario actualizarlo una vez al comienzo de cada año con la proyección del presente año (de tal manera que al menos quede la información del año anterior y el presente).
    Pasos: 1. Tomar el archivo “Proyeccion_cumpl_aportes.xlsx” (revisar punto 4 de la sección 7.1) y agregar la nueva información.
    2. Guardar como CSV.
    3. Reemplazar la tabla dwh.Proyeccion_cumpl_aportes en DWH subiendo la nueva información.

    Fuente 11: MetasCoberturaAfiliacion¶

    Esta fuente corresponde a la información de proyección de afiliación extraída de información suministrada por Cajamag. Este origen ya está en DWH con el nombre dwh.Meta_proyeccion_cobertura_afiliacion.
    Condición: Es necesario actualizarlo una vez al comienzo de cada año con la proyección del presente año (de tal manera que al menos quede la información del año anterior y el presente).
    Pasos: 1. Tomar el archivo “Meta_proyeccion_cobertura_afiliacion.xlsx” (revisar punto 4 de la sección 7.1) y agregar la nueva información.
    2. Guardar como CSV.
    3. Reemplazar la tabla dwh.Meta_proyeccion_cobertura_afiliacion en DWH subiendo la nueva información.

    Opción 2: Preparación automática (Python)¶

    Fuentes: ZoneDimCoor, Apo_Proy y MetasCoberturaAfiliacion¶

    Estas fuentes (Fuente 5, 6, 8 y 11) se pueden subir a DWH utilizando el script “main_conection_Dim_OP.py”, el cual se encuentra explicado en la siguiente matriz (revisar punto 7 de la sección 7.1):

    Tipo Objeto Nombre Objeto ETL/Consulta Anterior
    Script Python main_conection_Dim_OP_F N/A

    Descripción del objeto y cambio

    El script permite cargar a la base de datos DWH todas las tablas paramétricas de fuentes que lo necesitan para el modelo analítico. Esto en el caso que estos archivos se administren en formato Excel. Es necesario cambiar la ruta donde están los archivos en la línea 42.

    1. El script carga las librerías necesarias para ejecución:
      • Sqlalchemy versión 1.4.39
      • Pandas versión 2.0.3
      • Pymysql versión 1.4.6
    2. Crea la conexión a la base DWH.
    3. Lee el archivo de las fuentes y realiza el proceso indicado en la preparación con MySQL Workbench para la opción 1 de forma automática cada vez que se ejecute y se quiera actualizar las tablas (solo hay que cambiar la ruta).
    4. Lee el archivo de inventario de permisos y accesos y realiza el proceso indicado en la preparación con MySQL Workbench de forma automática.

    Fuente 7: Madurez¶

    Esta fuente corresponde a la información de la calificación de madurez empresarial que se realizará en el archivo “Formato Evaluación Madurez Modelo Empresarial.xlsx”. Sí es necesario actualizarlo. Se recomienda actualizarlo una vez al comienzo de cada semestre. Seguir los siguientes pasos:

    1. Tomar el archivo “Formato Evaluación Madurez Modelo Empresarial.xlsx” (revisar punto 4 de sección 7.1) y agregar la calificación.
    2. Esto solo es necesario hacer esto una sola vez: Abrir el tablero y entrar a transformación de datos.
    3. Seleccionar a la izquierda la tabla de Madurez.
    4. A la derecha seleccionar “Origen” y cambiar la “Ruta de acceso del archivo” por el lugar donde está ubicado el formato.
    5. A la izquierda, oprimir Cerrar y aplicar y luego de que cargue la tabla, oprimir Archivo -> Guardar.
