    Transformaciones Intermedias y Satisfacción¶

    Transformaciones Intermedias¶

    Para las fuentes marcadas en la tabla 2 como “Sí” para la pregunta ¿Requiere Transformación Intermedia?, es necesario aplicar el procesamiento intermedio de transformaciones. Para esto, se listarán los pasos a ejecutar para cada una de las fuentes. Así mismo, se plantea la opción de Python para realizar las transformaciones.

    Fuente 1: Periodos¶

    Esta fuente corresponde a la lista de periodos que se tomarán para el análisis en el modelo. Para la correcta ejecución seguir los siguientes pasos:

    1. Desde el servidor de Minerva, enviar la consulta que se encuentra en el archivo “ETL_AfiliaciónP1_Periodos.sql” a la base de pruebas dwh con el nombre “op_Periodos”.

    Fuente 12: Historico¶

    Esta fuente corresponde a la información de afiliaciones a través del tiempo de empresas, trayendo también información sociodemográfica. Para la correcta ejecución seguir los siguientes pasos:

    1. Desde el servidor de Minerva, enviar la consulta que se encuentra en el archivo “ETL_AfiliaciónP1_Empresas.sql” a la base de pruebas dwh con el nombre “op_EmpresasSubsi02_v2”.
    2. Desde la base de prueba dwh, enviar la consulta que se encuentra en el archivo “ETL_AfiliaciónP2_Empresas.sql” a la misma base de pruebas dwh con el nombre “op_sec_EmpresasSubsi02_v2”.

    (Opción) Preparación Automática (Python)¶

    Fuente 1 y 12: Periodos e Historico¶

    Estas fuentes se pueden subir a dwh con el script “main_conection_FactAfil_OP.py” explicado en la siguiente matriz:

    Tipo Objeto Nombre Objeto ETL/Consulta Anterior
    Script Python main_conection_FactAfil_OP N/A

    Descripción del Objeto y Cambio

    El script permite cargar a la base de datos DWH todas las tablas paramétricas de fuentes que lo necesitan para el modelo analítico. Esto en el caso que estos archivos se administren en formato Excel.

    1. El script carga las librerías necesarias para ejecución:
      • Sqlalchemy versión 1.4.39
      • Pandas versión 2.0.3
      • Pymysql versión 1.4.6
    2. Crea la conexión a la base DWH.
    3. Lee el archivo las fuentes y realiza el proceso indicado en la preparación con MySQL Workbench para de forma automática actualizar las tablas cada vez que se ejecute.
    4. Lee el archivo de inventario de permisos y accesos y realiza el proceso indicado en la preparación tradicional de forma automática.

    Fuente 20: Satisfaccion¶

    Esta fuente corresponde al formato planteado para consolidar las encuestas de satisfacción de los diferentes servicios. Para esto, se plantea la estructura de la tabla dwh.op_Satisfacción.
